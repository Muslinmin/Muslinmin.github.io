Project 1: Lite Kit, Wireless Controlled Mobile Robot

The wireless mobile robot system moves according to key commands that are transmitted from the computer through using UART transmission with XBee Wireless Module.

Robot stops when the edge of the table is detected

-

The robot uses Time of Flight Sensor to sense the distance between the table surface, and the floor, which is a trigger that will stop the robot from moving futher from the edge of the table.

The robot stops when an object is detected at both sides front and back

-

The robot uses an ultrasonic sensor, which detects the distance and stops when the distance between the robot and the object reaches a certain range.

Robot Basic Movement Control

-

Before the robot is integrated with the wireless communication functionality, the robot performs basic movement as a test with prescribed movements hardcoded into the Parallax board microcontroller.

Adaptive to new programming language

The microcontroller board used has a multicore functionality, which processes different programs concurrently within the same board. Unlike Arduino, which utilizes single core processing, this board can read a sensor while receiving wireless commands continously and running the motor all concurrently. The board has a language of its own, called SPIN. Unlike C and C++, which is used in other areas not limited to embedded programming, this language was created specificially for the board, which has its syntatical rules and how the programmer should program the board in such a way that can concurrently operate and retrieving data such as distance and commands all at the same time, to command the robot's movement.
